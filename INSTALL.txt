Система управления движением денежных средств (ДДС)
===========================================

1. Требования к системе
----------------------
- Python 3.8 или выше
- pip (менеджер пакетов Python)
- Git (для клонирования репозитория)

2. Установка
-----------
1. Клонируйте репозиторий:
   git clone https://github.com/shorakhmatov/Web_server_test.git
   cd Web_server_test

2. Создайте виртуальное окружение:
   python -m venv venv

3. Активируйте виртуальное окружение:
   - Windows:
     venv\Scripts\activate
   - Linux/Mac:
     source venv/bin/activate

4. Установите зависимости:
   pip install -r requirements.txt

3. Настройка базы данных
----------------------
1. Примените миграции:
   python manage.py migrate

2. Создайте суперпользователя:
   python manage.py createsuperuser

3. Загрузите начальные данные:
   python manage.py populate_initial_data

4. Запуск приложения
------------------
1. Запустите сервер разработки:
   python manage.py runserver

2. Откройте в браузере:
   http://127.0.0.1:8000/

5. Структура системы
------------------
1. Основные страницы:
   - Список транзакций: /
   - Создание транзакции: /transaction/create/
   - Редактирование транзакции: /transaction/<id>/edit/
   - Удаление транзакции: /transaction/<id>/delete/
   - Управление справочниками: /references/

2. Административная панель:
   - URL: http://127.0.0.1:8000/admin/
   - Логин: созданный при установке суперпользователь

6. Основные функции
-----------------
1. Управление транзакциями:
   - Создание новых транзакций
   - Просмотр списка с фильтрацией
   - Редактирование существующих
   - Удаление с подтверждением

2. Фильтрация транзакций по:
   - Дате (период)
   - Статусу
   - Типу операции
   - Категории
   - Подкатегории

3. Управление справочниками:
   - Статусы
   - Типы операций
   - Категории
   - Подкатегории

7. Устранение проблем
-------------------
1. Ошибка "No such table":
   - Выполните: python manage.py migrate

2. Ошибка при создании транзакции:
   - Проверьте, что загружены начальные данные
   - При необходимости: python manage.py populate_initial_data

3. Не работает фильтрация категорий:
   - Очистите кэш браузера
   - Проверьте консоль браузера на наличие ошибок JavaScript

8. Обновление системы
-------------------
1. Получите последние изменения:
   git pull

2. Обновите зависимости:
   pip install -r requirements.txt

3. Примените миграции:
   python manage.py migrate

9. Резервное копирование
----------------------
1. Данные SQLite:
   - Скопируйте файл db.sqlite3

2. Медиафайлы (если есть):
   - Скопируйте содержимое папки media/

10. Безопасность
--------------
1. В продакшене:
   - Измените SECRET_KEY в settings.py
   - Установите DEBUG = False
   - Настройте ALLOWED_HOSTS
   - Используйте HTTPS
   - Регулярно обновляйте зависимости

11. Поддержка
-----------
При возникновении проблем:
1. Проверьте логи сервера
2. Проверьте консоль браузера
3. Убедитесь, что все миграции применены
4. Проверьте версии установленных пакетов
